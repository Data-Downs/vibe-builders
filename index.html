An AI agent is a program that has a brain — an LLM (large language model) like Claude — and a job. You describe the job. Claude Code helps you build the program. The result is something that talks, thinks, reasons, and acts.

Before you begin

Four accounts.
Fifteen minutes.
Then you're ready.

Before touching the Terminal or writing a single line of code, you need to set up four accounts. All of them are free (or have a free tier). Do these in order. Keep a note of your usernames and passwords somewhere safe.

A

Anthropic Account

console.anthropic.com

Anthropic makes Claude — the AI brain behind everything you'll build. You need an account to use Claude Code and to get an API key (the secret key that lets your apps talk to Claude's intelligence).

  1. Go to console.anthropic.com
  2. Click Sign up and create an account
  3. Once signed in, go to API Keys in the left menu
  4. Click Create Key — give it a name like "my-first-app"
  5. Copy the key and save it somewhere safe — you'll only see it once
About cost

Anthropic gives you a small amount of free credit to start. For learning and building small projects, this goes a long way. You can add a card later if you need more — but don't worry about this now.

B

GitHub Account

github.com

GitHub is where your code lives. It's free, it's permanent, and it keeps a full history of everything you've ever built. Every time something works, you save it here. Think of it as a very smart, very safe backup for your work.

  1. Go to github.com
  2. Click Sign up
  3. Choose a username — this will be part of your project URLs, so pick something you're happy with
  4. Select the Free plan
  5. Verify your email address
C

Cloudflare Account

cloudflare.com

Cloudflare takes the app you've built on your laptop and puts it live on the internet — with a real URL you can share with anyone. The free tier is generous enough for everything you'll build while learning.

  1. Go to cloudflare.com
  2. Click Sign up — select the Free plan
  3. Verify your email address
  4. That's it for now — you'll come back to this in Step 6
D

Node.js — installed on your Mac

nodejs.org

Node.js isn't an account — it's a tool you install on your Mac. It's what allows your computer to run JavaScript-based code locally. Claude Code needs it to work. You install it once and never think about it again.

  1. Go to nodejs.org
  2. Download the version labelled LTS (Long Term Support — the stable, recommended one)
  3. Open the downloaded file and follow the installer
  4. To check it worked, open Terminal and type: node --version
  5. If you see a number (like v20.11.0), you're done

Getting started

The path. Step by step.

Follow these in order. Don't skip ahead. Each step builds on the last. And remember — Claude Code is with you the entire time. If something doesn't work, ask it why.

1

Sort out your folder — before anything else

Before you install a single thing, you need to know where your work will live. On a Mac, the default is to save everything to iCloud. This causes problems with the tools we're using. We need a folder that lives locally on your Mac — not in iCloud.

⚠ The iCloud Problem

iCloud syncs your files to Apple's servers. Claude Code and the Terminal work better with files that stay on your actual machine. If you skip this step, you'll hit confusing errors later that are hard to diagnose.

How to create a local folder:

Open Finder. In the menu bar at the top of your screen, click Go → Home. You'll see your home folder. Create a new folder here — call it something like Projects. This folder lives on your Mac, not in iCloud. This is where everything goes.

# Later, in the Terminal, you'll navigate here like this: cd ~/Projects
2

Open the Terminal. Don't panic.

Press ⌘ + Space to open Spotlight. Type Terminal. Hit Enter. A black (or white) window opens with a blinking cursor. This is the Terminal. It's waiting for you to type a command.

The Terminal is how you talk to your Mac directly — without icons, without clicking. It feels strange at first. Then it feels like a superpower.

The Terminal is not like normal text editing

You cannot click to place your cursor in the middle of a line. You cannot select a section and delete it. You navigate with the ← → arrow keys and delete with the backspace key. One character at a time. This is not a bug. It's just how it works.

The one thing that will save you the most time: Tab autocomplete. Start typing a folder or file name and press Tab — the Terminal will finish it for you. This is magic and you'll use it constantly.

See the full Terminal cheat sheet below — bookmark it, print it, tattoo it somewhere.

3

Install Claude Code

Claude Code is the AI that will build with you. It runs inside your Terminal and responds to plain English instructions. You type what you want. It writes the code.

Before you can install Claude Code, your Mac needs two things: Node.js (a behind-the-scenes tool that runs code) and npm (a package manager — it's how you install tools like Claude Code). Don't worry about understanding these fully. Just install them.

# Step 1: Install Node.js from nodejs.org — download the LTS version # (LTS means Long Term Support — the stable, recommended one) # Step 2: Once Node.js is installed, install Claude Code: npm install -g @anthropic-ai/claude-code # Step 3: Navigate to your Projects folder: cd ~/Projects # Step 4: Start Claude Code: claude

The first time you run it, Claude Code will ask you to sign in with your Anthropic account. Do that. Then you'll see a prompt. Type something. It'll respond. You're in.

4

Set up GitHub — your safety net

GitHub is where you save your work. Not just save — version it. Every time you make something work, you commit it to GitHub. If something breaks later, you can always go back. Think of it as an undo button with a memory.

Go to github.com and create a free account. Then install the GitHub command line tool:

# Install GitHub CLI (command line interface) brew install gh # If brew isn't installed, install it first: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Sign in to GitHub from your Terminal: gh auth login

Once you're set up, you'll commit your work regularly. Claude Code will tell you when it's a good time. The habit to build: whenever something works, commit it.

# The three commands you'll use constantly: git add . git commit -m "describe what you just did" git push
5

Run your first app locally

Before putting anything on the internet, you run it on your own laptop. This is called running it "locally". Your laptop becomes a mini server. You open a browser and go to a local address — usually localhost:3000 — and your app appears.

# Most apps start with one of these: npm run dev # or npm start

When you see something like Local: http://localhost:3000 in your Terminal — open that address in your browser. There's your app. Running on your laptop. This is the moment.

6

Deploy to Cloudflare — make it real

Once your app works locally, you put it on the internet. Cloudflare Pages takes your GitHub repository and turns it into a live website — with a real URL you can share with anyone. It's free. It takes about five minutes.

  1. Go to pages.cloudflare.com and create a free account
  2. Click Create a project → Connect to Git
  3. Select your GitHub repository
  4. Cloudflare will ask how to build it — Claude Code will tell you the right settings
  5. Click Deploy

Within minutes, Cloudflare gives you a URL. Your app is live. Anyone in the world can use it. That feeling doesn't go away.

API Keys & Environment Variables

If your app uses Claude's AI (it will), you'll have a secret API key. Don't put this in your code directly. Cloudflare has a section called Environment Variables where you store secrets safely. Claude Code will walk you through this.

⚠ The quirks nobody tells you

You cannot click to move your cursor. Unlike a normal text editor, clicking in the middle of a line does nothing. Use the ← and → arrow keys to navigate. Use backspace to delete. One character at a time.

Tab autocomplete is your best friend. Start typing a folder or file name and press Tab — the Terminal will complete it for you. If nothing happens, press Tab twice to see all possibilities. Use this constantly.

↑ arrow = previous command. Press the up arrow to cycle through commands you've already typed. You'll use this constantly instead of retyping things.

Nothing happens when you type a password. If the Terminal asks for your Mac password, type it and press Enter — even though nothing appears on screen. It's reading your input, just not showing it.

How to build

From idea to working app.
The process.

Claude Code isn't just a code writer. It's a thinking partner. Before it writes a single line, it will plan with you. This is the most important part — and most people skip it.

01

Write your brief

Describe what you want to build in plain English. Don't worry about technical terms. Don't worry about being precise. Describe the experience you want someone to have. Who is this for? What does it do? How does it feel to use it?

The brief doesn't have to be right. It just has to be honest. Claude Code will ask you questions if it needs more.

The ingredients of a good brief:
  • What it is — describe it like you'd describe it to a friend
  • Who uses it — one specific person, not "everyone"
  • What they do with it — the main thing, the core interaction
  • What it feels like — warm? sharp? playful? serious?
  • What it should NOT do — constraints are as useful as requirements
02

Start in planning mode — don't build yet

This is the most important thing in this entire guide. When you first start Claude Code, add these words to your brief:

# Start Claude Code in your project folder: claude # Then paste your brief, followed by this instruction: Before writing any code, I want you to think through this carefully and give me a detailed plan. What will you build? What are the main components? What decisions are you making and why? Do not start coding until I approve the plan.

Claude Code will give you a clear breakdown of what it intends to build. Read it. Question it. Push back if something doesn't sound right. This is your chance to steer before anything is built.

Why this matters

If you let Claude Code start building immediately, it will make dozens of decisions you didn't know were being made. Planning mode surfaces those decisions. It's the difference between directing a project and inheriting one.

03

Review and approve the plan

Read the plan carefully. Ask yourself: does this match what I imagined? Are there things I didn't think about that I should have an opinion on? Are there things in the plan I don't understand?

Ask Claude Code to explain anything you don't understand. Change anything that doesn't feel right. When the plan matches your vision, say:

That looks good. Go ahead and build it.

Then watch. It will build quickly. When it's done, it will usually tell you how to run it locally — type the command it gives you, open your browser, and see what you've made.

04

Use it. Break it. Improve it.

The first version is never the final version. Use the app. Notice what doesn't feel right. Notice what's missing. Then go back to Claude Code and describe what you want to change — just like the original brief. Plain English. Specific and honest.

# Examples of good enhancement prompts: "The tone feels too formal. I want it to feel more like a conversation with a smart friend." "When I ask it something it doesn't know, it makes things up. I want it to say it doesn't know instead." "I want to add a way to save the conversations so I can come back to them."

Each enhancement is a new brief. Each new brief is another loop through this process. This is how software is built — not in one big push, but in small, considered iterations.

Commit before every change. Always run git add . && git commit -m "working version before changes" before asking Claude Code to modify something. That way, if it breaks, you can always go back.

A tip about starting

Your first idea will probably change. That's the point.

You might start wanting to build an archivist — something that organises your notes and writing. Then realise everything you know is in your head, not on paper. So you flip the brief: instead of an archivist, you build an interviewer that asks you questions to draw out what you know.

Same instinct. Different form. The brief evolved in minutes. That's the process. Start with what feels right, and let the act of building teach you what you actually want.
